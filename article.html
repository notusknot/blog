How I made my website

I've wanted to make a website for a long time, and after some learning and practice, that is finally a reality :). In my very first blog post, I'll go over the philosophy and ideas for this website, how I actually made it, and how you can make your own.

Philosophy

First and foremost, I wanted this site to be minimal. I'm a fan of minimalism in real life, and even more so when it comes to technology. As the ultimate challenge (it's not actually that difficult), I decided to make my site with absolutely 0 JavaScript and no external pings to other websites. I did this for a few reasons:

  1. No JS makes the site load faster and it means people who disable JS can have the same experience
  2. No external pings makes the site load faster
  3. I don't like programming in JS
  4. It's a fun challenge

So, how did I actually accomplish this?

Well, vanilla HTML and CSS are surprisingly powerful on their own. You can easily create a nice, fast and secure interface with HTML and CSS. Now don't get me wrong; JavaScript is a very useful language, but it's simply not necessary for a site like mine.

The only technical challenge I had with this approach was syntax highlighting for code blocks. Most options out there are done with JS, which is probably the best way to do it but I needed to stick to my challenge. You can always add an insane amount of <span> tags everywhere and style them with CSS, but that's far from the best option... or is it?

Enter pygments. Pygments is a Python program for doing generic syntax highlighting across several hundred different languages. The best part: you can export it to nicely formatted HTML for ease of use with CSS! For example:

with import <nixpkgs> { };

mkShell {
    buildInputs = [
        python39Packages.pygments
        xclip
    ];
}

If you turn off JavaScript in your browser, you can see you still get syntax highlighting. Isn't that awesome?

Now, to use Pygments I decided to write a quick mapping in Neovim for a simple task. Here is the code snippet:

map('n', '<C-s>', ':!xclip -sel c -o | pygmentize -f html | xclip -sel c <CR> <CR>', options)

If you have a Lua configuration for Neovim, you can drop that in. When you press Ctrl + S, it will read the code from your clipboard, highlight it and convert to HTML, and then put it back in your clipboard so you can paste it wherever you want.

How I made the blog

Well now that we have all that out of the way, how did I actually create the blog? It's pretty simple. Since I followed such a simple philosophy, it makes the creation and maintenance of the site magnitudes easier. Other than my special ingredient, (which I will get to soon), it follows the basic structure of an HTML website.

To make the development of my blog easier, I created a little script called "bmanager". Bmanager basically does all of the repetitive work for me so I can just focus on writing. It's in the repo for this website, if you would like to check it out.
